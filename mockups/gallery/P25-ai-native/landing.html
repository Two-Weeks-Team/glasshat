01 · synthesize

Rubric becomes data, not opinion.

A RubricSynthesizer agent parses the evaluator's official rules into a typed rubric with weights, criteria, evidence requirements, and a hash. Same engine, different rubric, different (correct) score.

tool qdrant.upsert  · collection=rubric_definitions
  rubric_id=qdrant-vsd-2026 · weights_hash=0x9a4f2c…
  criteria=[func, originality, ux, presentation]
02 · panel

Six perspectives, in parallel, with tools.

ADK ParallelAgent[6] dispatches the Six Thinking Hats. White grounds with Vertex search; Black must cite a precedent or the run aborts via a Plugin guard.

ParallelAgent(name="HatsPanel", children=[
  WhiteAgent, RedAgent, YellowAgent,
  BlackAgent(must_cite_precedent=True),
  GreenAgent, BlueSynthesisAgent,
])
03 · detect

Triple-redundant detection.

Phoenix Online Eval (LLM-as-judge) + Phoenix Custom Evaluator (deterministic) + Black hat counter-claim all converge on yellow.A1 over_confident. Any one fires the AuditLoop.

eval.calibration.label = "over_confident"
  yellow.score = 9.0 · evidence_depth = 0.31
  threshold = 0.40 · violated by 0.09
04 · consult

Phoenix MCP. Four calls in parallel.

The PhoenixConsultantAgent queries the calibration experiment, fetches span annotations, and pulls 3 anchor profiles. Then Qdrant's Recommendation API picks the anti-pattern negatives.

mcp.call get-experiment-by-id("glasshat-calibration-v1")
  filter={hat:"yellow", criterion:"A1", depth_bkt:"<0.4"}
  → {mean_delta: -1.2, n: 14, CI: [0.9, 1.5]}
05 · correct

Score self-corrects, on screen.

ScoreCalibrationAgent applies clip(score − 0.8·Δ, p25, p75) with a ±2.0 cap, writes back to session state, signals loop convergence. The number animates. The 3D graph reshapes.

session.state["hat_yellow_score_a1"] = 9.0 → 7.6
tool_context.actions.escalate = True
→ SSE { type: "score_corrected", anchors: [3] }
